AI提示词写作技巧

掌握提示词工程，让AI更好地理解你的需求

什么是提示词（Prompt）

提示词是你向AI模型输入的指令或问题，它直接决定了AI输出的质量和效果。

一个好的提示词能够：

  • 明确任务目标 - 告诉AI你想要什么
  • 设定角色和背景 - 让AI以特定身份回答
  • 定义输出格式 - 控制AI的回复形式
  • 提供示例和约束 - 确保输出符合预期

提示词设计原则

原则1：明确具体

模糊的指令导致模糊的结果。越具体，效果越好。

❌ 不好："写一篇关于AI的文章" ✅ 好："作为一名科技记者，写一篇800字关于生成式AI对职场影响的深度报道，要求包含3个真实案例"
原则2：设定角色

让AI扮演特定角色，可以获得更专业的回答。

❌ 不好："解释什么是机器学习" ✅ 好："你是一名大学计算机教授，用通俗的语言向大一新生解释什么是机器学习"
原则3：定义格式

明确输出格式，让AI按照你想要的结构回复。

✅ 示例："请用表格形式列出，包含：产品名称、价格、优缺点"
原则4：分步指导

复杂任务可以拆分成多个步骤来完成。

✅ 示例："第一步：分析用户需求；第二步：提出3个方案；第三步：比较各方案优劣"

提示词基本结构

标准提示词公式

角色设定 + 任务描述 + 输出要求 + 约束条件

结构详解：
  1. 角色设定 - 让AI扮演什么身份（专家、顾问、创作者等）
  2. 任务描述 - 具体要完成什么任务
  3. 输出要求 - 输出的格式、长度、语言风格等
  4. 约束条件 - 有什么限制或注意事项
完整示例
【角色设定】你是一位资深的产品经理，拥有10年互联网产品经验。

【任务描述】我想开发一个在线学习平台，请帮我分析核心功能模块。

【输出要求】
1. 列出5个核心功能模块
2. 每个模块包含：功能描述、用户价值、优先级
3. 用表格形式呈现
4. 最后给出一个MVP开发建议

【约束条件】
- 目标用户：职场人士
- 预算有限，优先核心功能
- 考虑移动端优先

实用示例

你是一位优秀的广告文案策划师。

请为一款新型降噪耳机撰写：
1. 产品标题（20字以内）
2. 产品卖点（3个，每点不超过30字）
3. 电商详情页开头文案（100字左右）

目标用户：经常出差的商务人士
语言风格：专业、有品质感

你是一位资深的前端开发工程师，精通Vue3和TypeScript。

请帮我实现一个用户登录表单组件，要求：

技术栈：Vue3 + TypeScript + Element Plus
功能点：
1. 手机号验证（中国手机号格式）
2. 密码强度检测
3. 发送验证码倒计时
4. 表单提交和错误处理
5. 记住密码功能

输出格式：
- 完整的Vue组件代码
- 代码注释
- 使用说明

你是一位数据分析师，擅长从复杂数据中提取洞察。

我有以下销售数据：
- 月度销售额：1月50万，2月65万，3月72万，4月80万，5月68万，6月90万
- 主要产品：A(30%)、B(45%)、C(25%)
- 客单价：平均2500元

请帮我分析：
1. 销售趋势分析
2. 产品表现评估
3. 业务建议

输出格式：用Markdown格式，包含数据可视化的文字描述

高级技巧

迭代优化

提示词不是一次写好的，需要根据AI的输出不断调整优化：

  1. 先用基础版本测试
  2. 分析输出中的问题
  3. 针对性修改提示词
  4. 再次测试验证
  5. 重复上述步骤
对话式提示

将复杂任务拆分成多轮对话：

第一轮："分析这个需求的核心要点"

第二轮："基于分析，提出3个方案"

第三轮："选择方案2，详细实现"

Few-Shot提示

提供少量示例让AI学习你的期望：

将用户评价转换为情感标签：

示例1："产品很好用" → 正面

示例2："发货太慢了" → 负面

现在分析："客服态度一般，但产品质量不错"

思维链（CoT）

让AI展示推理过程，提高复杂任务准确率：

关键短语：

  • "一步一步来分析"
  • "先思考，再回答"
  • "解释你的推理过程"
  • "详细说明每个步骤"

常见错误

类型 错误示例 问题分析
模糊 "帮我写点东西" 缺少具体的主题、格式、长度要求
信息不足 "分析这个产品" 没有提供产品的具体信息
任务过载 "帮我开发一个完整的APP" 任务过于复杂，应该拆分成多个步骤
缺少示例 "用特定风格写作" "特定风格"太模糊，应该给出示例
语言问题 混用中英文术语 保持语言一致性，避免AI混淆
